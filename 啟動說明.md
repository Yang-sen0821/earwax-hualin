# 樺林美學 E•ar Wax — 啟動說明

> 本系統技術棧：Flask + Flask-SQLAlchemy + Supabase(PostgreSQL)。
> 部署比照泰安：本機開發 → **共用 MUSEN-SAAS（earwax schema 隔離）** → Render。
> 重要原則：**密碼一律放環境變數（.env / Render Dashboard），程式碼不寫死。**
>
> 🔒 **資料隔離設計**：本系統與泰安共用同一顆 Supabase（MUSEN-SAAS），但所有表
> 一律建在獨立的 PostgreSQL schema 「**earwax**」下，與泰安的 `public` schema 完全
> 隔開。即使兩邊都有 `customers` 表也不會互相污染。`init_db.py` 與 app 啟動時會自動
> `CREATE SCHEMA IF NOT EXISTS earwax`，不需手動建。**不需要、也不要另開新 Supabase 專案。**

- Python 3.9 以上（開發機已驗證 3.9）。
- 套件：見 `requirements.txt`。安裝：
  ```bash
  pip install -r requirements.txt
  ```

## 2. 環境變數設定

### 2-1. 複製範本
把 `.env.example` 複製成 `.env`，填入真實值。`.env` 已列入 `.gitignore`，不會被 commit。

```bash
cp .env.example .env
```

### 2-2. 必填變數
| 變數 | 說明 | 範例 |
|------|------|------|
| `SECRET_KEY` | Flask session 加密金鑰，正式環境改成隨機長字串 | `任意亂數長字串` |
| `DATABASE_URL` | Supabase PostgreSQL 連線字串 | `postgresql://postgres:密碼@db.xxxx.supabase.co:5432/postgres` |

### 2-3. 選填（init_db.py 種子帳密來源）
| 變數 | 預設值 | 說明 |
|------|--------|------|
| `OWNER_USER` | `owner` | 老闆帳號，可看後台儀表板 |
| `OWNER_PASS` | `OwnerPass@2024` | 老闆密碼，**正式請改強密碼** |
| `STAFF_USER` | `staff` | 員工帳號，只能做 4 步登錄與查庫存 |
| `STAFF_PASS` | `StaffPass@2024` | 員工密碼，**正式請改強密碼** |

> ⚠️ 預設密碼僅供首次建表方便，森哥務必在 `.env` 改成自己的密碼後再跑 `init_db.py`。

## 3. 取得 MUSEN-SAAS 連線字串（共用，不另開專案）

本系統掛在既有的 **MUSEN-SAAS** Supabase（泰安/景觀也在上面），靠 `earwax` schema 隔離。
不需要新建專案，只要拿到 MUSEN-SAAS 的連線字串：

1. 登入 <https://supabase.com> → 開啟 **MUSEN-SAAS** 專案。
2. 左側 **Project Settings → Database → Connection string → URI**，
   複製 `postgresql://postgres:[YOUR-PASSWORD]@db.xxxx.supabase.co:5432/postgres`。
3. `[YOUR-PASSWORD]` 換成 MUSEN-SAAS 的 Database Password，整串貼進 `.env` 的 `DATABASE_URL`。

> ✅ 安全性：本系統只會在 `earwax` schema 內建表/讀寫，碰不到泰安的 `public` 資料。
> 但連線字串等於整顆 DB 的鑰匙，請當機密保管，勿外流、勿寫進程式碼。
>
> ⚠️ IPv4 相容（泰安踩過的雷）：**direct connection（`db.xxxx.supabase.co`）是 IPv6-only**，
> Render 與多數 IPv4 環境連不到，泰安當初就卡在這。**一律用 Pooler（`...pooler.supabase.com`）**：
> - 首選 **Session pooler（port 5432）**：長駐 Flask 最穩，IPv4 OK。
> - 次選 **Transaction pooler（port 6543）**：IPv4 也 OK，但 transaction 模式需在連線字串
>   加 `?prepared_statement_cache_size=0`（或等效設定）避免 prepared statement 衝突。

## 4. 建表 + 寫入種子資料（init_db.py）

確認 `.env` 已填好後，執行：

```bash
python init_db.py
```

會做的事（可重複執行，已存在就略過，不會重複建立）：
- 先 `CREATE SCHEMA IF NOT EXISTS earwax`（與泰安隔離）。
- 在 `earwax` schema 下建立全部資料表（earwax.users / earwax.products / earwax.params /
  earwax.customers / earwax.voucher_inventory / earwax.voucher_purchases /
  earwax.consumables / earwax.consumable_purchases / earwax.service_records /
  earwax.service_consumables）。
- 種子：
  - owner / staff 兩組帳號（密碼取自 `.env`，存的是 hash）。
  - 愛啪啪 Product + Params（會員/非會員售價、變動成本 2620、股東回收 10 等）。
  - 美容券庫存初始 0 張（之後用「補充美容券」入庫）。
  - 消耗品種子（qty 0、unit_cost 0 待後補）：針筒、耳塞、安瓶、洗卸品、洗臉巾、防曬。
  - 儀器設備種子（固定資產，只記清單）：導入儀、氣壓儀器、頭皮臉部偵測儀、台車、破壁機。

執行後會印出各表筆數，確認 users=2、products=1、params=1、
voucher_inventory=1、consumables=11（6 消耗品 + 5 儀器）即代表種子完成。

## 5. 本機啟動

### 方式 A：直接跑 app.py（最簡單）
`app.py` 不會自動載入 `.env`，請先把變數匯入 shell：

PowerShell：
```powershell
$env:SECRET_KEY = "本機測試金鑰"
$env:DATABASE_URL = "postgresql://postgres:密碼@db.xxxx.supabase.co:5432/postgres"
python app.py
```

bash：
```bash
export SECRET_KEY="本機測試金鑰"
export DATABASE_URL="postgresql://postgres:密碼@db.xxxx.supabase.co:5432/postgres"
python app.py
```

啟動後開 <http://localhost:5000>。

### 方式 B：flask run（自動讀 .env）
已安裝 `python-dotenv`，`flask run` 會自動載入同目錄 `.env`：
```bash
flask --app app run --debug
```

### 登入驗證
- 用 owner 帳密登入 → 看得到「後台儀表板」（回本進度、會員占比、明星耗材…）。
- 用 staff 帳密登入 → 直接進「新增施作」，進不了後台（會被導回）。

## 6. Render 部署

1. 把整個專案 push 到 GitHub repo（`.env` 不會被帶上，已被 gitignore）。
2. Render → New → **Web Service** → 連到該 repo。
3. Render 會讀到 `render.yaml` / `Procfile`：
   - Build：`pip install -r requirements.txt`
   - Start：`gunicorn app:app --bind 0.0.0.0:$PORT`
4. 在 **Render Dashboard → Environment** 手動加環境變數（**不要寫進 yaml**）：
   - `SECRET_KEY`（隨機長字串）
   - `DATABASE_URL`（同一組 Supabase 連線字串）
5. 首次部署後，需要對線上 DB 建表 + 種子。兩種做法擇一：
   - 本機把 `.env` 的 `DATABASE_URL` 指向同一個 Supabase，跑一次 `python init_db.py`（推薦，最直覺）。
   - 或在 Render 的 Shell 執行 `python init_db.py`。
6. 開 Render 給的網址，用 owner 帳號登入確認。

> 健康檢查端點：`/health`，回 `{"status":"ok"}` 代表 DB 連得上。

## 7. 預設 owner / staff 帳密來源

- 帳密**不寫死在程式碼**，由 `init_db.py` 從環境變數讀取：
  `OWNER_USER` / `OWNER_PASS` / `STAFF_USER` / `STAFF_PASS`。
- 未設定時的 fallback 預設值（僅供首次方便，正式務必改）：
  - owner：`owner` / `OwnerPass@2024`
  - staff：`staff` / `StaffPass@2024`
- 改密碼流程：改 `.env` 的對應變數 → 重跑 `init_db.py`。
  （註：`init_db.py` 對已存在帳號是「略過」，若要改既有帳號密碼，
  需先在 DB 刪該 user 再重跑，或日後做後台改密碼功能。）

## 8. 常見問題

- **啟動報 `RuntimeError: 環境變數 DATABASE_URL 未設定`**
  → `.env` 沒填或沒被載入。用方式 B（flask run）或先 export 變數。

- **連不上 Supabase / `unable to open database`**
  → 檢查 `DATABASE_URL` 密碼是否正確、host 是否為該專案、port 用 5432。

- **券庫存不足無法登錄非會員**
  → 正常擋件。先到「庫存 → 補充美容券」入庫後再登錄。

- **會員登錄沒扣券庫存**
  → 正確行為。會員自帶券，不扣店家券庫存；所選消耗品仍會扣（儀器設備不扣）。

- **dashboard 數字都是 0**
  → 尚無施作記錄，或日期篩選把資料濾掉了。先登錄一筆或清除篩選。
