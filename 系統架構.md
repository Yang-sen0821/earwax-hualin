# 樺林美學 E•ar Wax — 系統架構

> 本檔是**人看的系統地圖**（高層設計）。欄位/表/路由的權威定義在 `SPEC.md`，兩者衝突以 SPEC.md 為準。
> 變動時：先改 SPEC.md，再回來同步本檔。

---

## 1. 一句話定位
給樺林美學單店用的**代理盈虧 + 庫存 + 會員獲利**管理網站。員工用 4 步登錄每筆消費，老闆後台看回本進度與損益。技術與部署比照泰安專案。

---

## 2. 技術棧（比照泰安）
```
前端：Jinja2 模板 + Bootstrap 5(CDN) + Chart.js(CDN)
後端：Flask + Flask-SQLAlchemy（blueprints 分模組）
資料庫：Supabase (PostgreSQL)  ← 共用 MUSEN-SAAS，earwax schema 隔離（不與泰安撞表）
登入：Flask session + werkzeug 密碼雜湊
部署：Render（gunicorn）；設定全走環境變數
```

---

## 3. 模組地圖（blueprints）
```
app.py ──註冊──┬─ auth        /login /logout（session 存 role）
               ├─ service     /service/new（4步核心）、/service 列表
               ├─ customers   /customers、/customers/new
               ├─ inventory   /inventory（券庫存 + 消耗品庫存 + 儀器設備清單）
               ├─ purchases   /purchases/voucher（補券）、/purchases/consumable（補消耗品）
               └─ dashboard   /（老闆限定，回本與損益儀表板）

權限：login_required（全站）／owner_required（僅 dashboard）
       staff 登錄消費、查庫存；owner 另可看後台。
```

---

## 4. 資料模型（ER 概念圖）
```
User(帳號/role)                Product(愛啪啪) ──1:1── Params(定價/成本參數)
                                     │
Customer(會員/非會員) ──1:N── ServiceRecord(施作記錄=4步輸出) ──N:1── Product
                                     │  扣券(非會員) / 扣消耗品(ServiceConsumable 明細，可多項+贈品)
                                     ▼
        VoucherInventory(券庫存,張)        ◄──加── VoucherPurchase(補券)
        Consumable(消耗品/儀器,件)         ◄──加── ConsumablePurchase(補消耗品)
        ServiceRecord ──1:N── ServiceConsumable(施作耗材明細) ──N:1── Consumable
```
> 消耗品(category='consumable')施作扣用；儀器設備(category='equipment')固定資產不扣。
詳細欄位見 SPEC.md 第 2 節。

---

## 5. 核心流程：4 步登錄一筆消費
```
員工點「新增消費」
 ① 選代理產品（目前僅 愛啪啪）
 ② 選消費者（有則套用 / 無則當場新增）
 ③ 選身分：會員 or 非會員
 ④ 輸入：張數、實收金額(可折扣)、耗材明細(可多項消耗品，各填數量、各自可標贈品)
        │
        ▼ 後端送出時（不信前端，後端權威）
   ┌─ 算淨利：profit = 實收 − 變動成本2620×張數 + 店主股東回收10×張數
   ├─ 非會員 → 券庫存 −張數（不足則擋下提示）
   ├─ 會員   → 不扣券庫存（客自帶券）
   └─ 逐筆消耗品 → consumables.qty_on_hand −用量（僅 category='consumable'，含贈品；不足擋下）
        ▼
   寫入 ServiceRecord，更新庫存
```

---

## 6. 會員 vs 非會員（獲利差來源）
| 身分 | 券哪來 | 基準售價 | 基準淨利 | 對庫存 |
|------|--------|---------|---------|--------|
| 非會員 | 跟店家買 | 3980 | 1360(+店主股東10) | **扣券庫存** |
| 會員 | 自帶 | 3100 | 480(+店主股東10) | 不扣 |

差別只在售價，變動成本相同（2620/張）。非會員可給折扣 → 改實收 → 淨利自動重算。

---

## 7. 兩套獨立庫存
```
美容券庫存(張)   ── 補券(VoucherPurchase) 加 / 非會員消費 扣
消耗品庫存(件)   ── 補消耗品(ConsumablePurchase) 加 / 每次施作的消耗品(含贈品) 扣
```
消耗品(針筒/耳塞/安瓶/洗卸品/洗臉巾/防曬)管庫存施作扣用；儀器設備(導入儀/氣壓儀器/偵測儀/台車/破壁機)為固定資產，只記清單不逐次扣。

---

## 8. 後台儀表板指標（老闆限定）
- **愛啪啪回本**：累計淨利、剩餘回本、回本進度%（/142,000）
- **全店總覽**：愛啪啪單一主體損益（累計淨利、累計營收、客數）— 不含任何其他主體
- **會員/非會員**：人數、占比、各自累計淨利（比照 Excel 總覽）
- **明星耗材**：依 ServiceConsumable 加總各消耗品用量（含贈品），由多到少排名
- **回本週期**：依日期區間平均淨利/客 × 來客頻率，推估還需多久回本
- **期間篩選**：start / end

## 9. 部署拓樸
```
 員工/老闆瀏覽器 ──HTTPS──► Render(gunicorn+Flask) ──► MUSEN-SAAS Supabase
                                   │                      └─ schema: earwax（樺林專用）
                                   │                      └─ schema: public（泰安，互不干擾）
                              環境變數：SECRET_KEY / DATABASE_URL / 種子帳密
```
⚠️ 安全：
- DB 連線字串只走環境變數，原始碼不寫死密碼（修正泰安的壞示範）。
- 與泰安共用同一顆 Supabase（MUSEN-SAAS），但樺林所有表在 `earwax` schema，
  泰安在 `public`，兩者完全隔離，`customers` 等同名表不會互相污染。

---

## 10. 與泰安的關係
- **沿用**：Flask+SQLAlchemy+Supabase 棧、blueprints 結構、session 登入、補貨/扣庫存/儀表板的寫法。
- **新增**：權限分級（員工/老闆）、4 步消費登錄、雙庫存（券+消耗品）、消耗品/儀器分類、回本/折扣淨利邏輯。
- **共用但隔離**：與泰安共用 MUSEN-SAAS Supabase（統一管理、一份帳單，符合方案B「地基共用」），
  但樺林表全在 `earwax` schema，與泰安 `public` 完全隔離；Render 為獨立服務。
